# PECHUNIA 管理画面の使い方

PECHUNIA の管理画面は、**ニュース／アーティスト／オーディション情報**を
ブラウザ上で編集して、変更を GitHub に保存（=サイトに公開）する仕組みです。

公開サイトと同じドメインの **`/admin/`** からアクセスできます。

```
https://yamada41122.github.io/pechunia-web/admin/
```

---

## 仕組みの概要

```
[管理画面]
  │  GitHub API + あなたのトークン
  ▼
[GitHub のリポジトリ]
  │  data/news.json などを更新
  ▼
[GitHub Pages が自動再ビルド]
  │  数十秒〜2分
  ▼
[サイトに反映]
```

データ本体は `data/` フォルダの **3つのJSONファイル**：
- `data/news.json` — ニュース一覧
- `data/artists.json` — 所属アーティスト
- `data/audition.json` — オーディション情報

公開サイト側は、これらJSONをJavaScriptで読み込んで動的に表示しています。
管理画面で編集 → コミット されると、JSONファイルが書き換わり、
GitHub Pages が再ビルドして数分でサイトに反映されます。

## 初回セットアップ（スタッフ各自で1度ずつ）

### 1. GitHub の Personal Access Token を発行

各スタッフが自分のGitHubアカウントでトークンを発行します。
これにより「誰がいつ編集したか」がコミット履歴に残ります。

1. GitHub にログイン
2. https://github.com/settings/tokens?type=beta を開く
3. **Generate new token** をクリック
4. 設定：
   - **Token name**：`pechunia-admin`（任意の名前）
   - **Expiration**：90日〜1年 推奨
   - **Resource owner**：自分のアカウント
   - **Repository access**：
     - `Only select repositories` を選び、`pechunia-web` リポジトリを選択
     - （複数人で使う場合、リポジトリの所有者が各スタッフを Collaborator に追加しておく必要があります）
   - **Repository permissions**：
     - **Contents**：`Read and write`
     - **Metadata**：`Read-only`（自動）
5. **Generate token** をクリック
6. 表示された `github_pat_xxxx...` で始まるトークンをコピー
   **※ このページを閉じると二度と表示されないので、必ず安全な場所にメモ**

### 2. 管理画面にログイン

1. https://yamada41122.github.io/pechunia-web/admin/ を開く
2. フォームに入力：
   - **オーナー（GitHubユーザー名）**：`yamada41122`
   - **リポジトリ名**：`pechunia-web`
   - **Personal Access Token**：先ほどコピーしたトークン
3. **ログイン** をクリック

ログインに成功すると、ニュース管理画面が開きます。
トークンはお使いのブラウザに保存されるので、次回からは自動でログイン情報が入ります。

## 使い方

### News（ニュース）の編集

1. **News** タブ（既定で開いている）
2. **+ 新規ニュースを追加** ボタンで投稿
   - 日付：`2026.04.18` の形式
   - カテゴリー：LIVE / MEDIA / INFO の3つ
   - タイトル：本文
3. 並び替え：各行の **↑ ↓** ボタン
4. **編集** で内容変更、**削除** で削除
5. すべて整ったら **変更を保存** ボタン → GitHub にコミット

### Artists（アーティスト）の編集

1. **Artists** タブ
2. **+ 新規アーティストを追加** で登録
   - 表示順 No、カテゴリー、名前、イニシャル、カード色などを入力
3. 並び替え・編集・削除は News と同じ
4. **変更を保存** で反映

### Audition（オーディション）の編集

1. **Audition** タブ
2. キャッチコピー、リード文、締切表記、CTAボタンなどを直接編集
3. **募集要項**・**応募方法**：項目ごとに編集 / 追加 / 削除
4. **選考フロー**：4ステップの内容を編集
5. **スケジュール**：表のラベルと内容を編集
6. **変更を保存** で反映

## 反映されない/エラーが出るとき

### 「保存しました」と出たのにサイトに反映されない
GitHub Pages のビルドに**1〜2分かかります**。少し待ってからリロード（ブラウザの再読込ボタン）してみてください。
それでも反映されない場合は、リポジトリの **Actions** タブでビルド状況を確認できます。

### 401 / トークンが無効です
- トークンの有効期限が切れている
- トークンをコピー漏れしている
- → 新しいトークンを発行してログインし直してください

### 404 / リポジトリが見つかりません
- リポジトリ名（`pechunia-web`）かオーナー名のスペル違い
- トークンに当該リポジトリへの権限がない
- → トークン作成時の Repository access / Contents 権限を確認

### 「他の人と同時に編集していたデータが上書きされそう」
JSONファイルの **sha**（バージョン識別子）を見て GitHub が衝突を検出します。
他の人が先に保存していた場合は **保存失敗** のエラーになるので、ページをリロードして
最新を取得し直してから編集してください。

## トークンの管理について

- トークンは **発行した本人のブラウザ** にのみ保存されます
- 共有のPCで使う場合は使い終わったら **ログアウト** ボタンを押してください
- 退職・引き継ぎなどの際は GitHub の Settings → Personal access tokens で
  該当トークンを **Revoke**（無効化）してください

## ローカルでテストする（オプション）

開発者向け：
ローカルで管理画面を試す場合、ファイル直開きでは fetch が動かないので、
簡易サーバーを立ててください：

```bash
cd "/Users/yamadadaiki/Documents/Claude Code/pechunia_web"
python3 -m http.server 8000
# ブラウザで http://localhost:8000/admin/ を開く
```

ローカルから保存（コミット）するには、ログイン時にオーナー名とリポジトリ名を
手動で入力してください（ローカルでは自動検出できないため）。
